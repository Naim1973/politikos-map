# Server Setup Guide

Step-by-step guide to get the Politikos Map server running from scratch on a new machine.

## Prerequisites

- **Node.js 22+**
- **npm** (comes with Node.js)
- A **Convex** account — sign up free at [convex.dev](https://www.convex.dev/)

---

## Step 1: Install Dependencies

```bash
cd server
npm install
```

> **Version note:** `better-auth` is pinned to `1.4.9` because `@convex-dev/better-auth` requires `>=1.4.9 <1.5.0`. Do not upgrade `better-auth` without checking compatibility.

---

## Step 2: Initialize Convex

This is interactive — it will open your browser to log in and create a project.

```bash
npx convex dev
```

Follow the prompts:
1. Log in to your Convex account
2. Create a new project (or select an existing one)
3. Wait for it to push the schema and functions

**Keep this terminal running.** Convex dev watches for file changes and auto-deploys.

Once it finishes, it creates a `.env.local` file with:
```
CONVEX_DEPLOYMENT=dev:<your-deployment-name>
CONVEX_URL=https://<your-deployment>.convex.cloud
```

---

## Step 3: Add Environment Variables

### 3a. Set Convex deployment variables

In a **separate terminal** (keep `npx convex dev` running), set the auth secret and site URL on your Convex deployment:

```bash
cd server

# Generate and set a random secret (must be 32+ characters)
npx convex env set BETTER_AUTH_SECRET $(node -e "console.log(require('crypto').randomBytes(32).toString('base64'))")

# Set the Hono server URL
npx convex env set SITE_URL http://localhost:3001
```

> **Why set these on Convex?** Better Auth runs inside Convex as a component. It reads `BETTER_AUTH_SECRET` and `SITE_URL` from the Convex runtime environment, not from your local `.env.local`.

### 3b. Add CONVEX_SITE_URL to .env.local

Open `.env.local` (auto-generated by Convex in Step 2) and add the `CONVEX_SITE_URL` line. Your deployment name is the same one from `CONVEX_URL` but ending in `.site` instead of `.cloud`:

```
# Already generated by Convex:
CONVEX_DEPLOYMENT=dev:adjective-animal-123
CONVEX_URL=https://adjective-animal-123.eu-west-1.convex.cloud

# Add this manually:
CONVEX_SITE_URL=https://adjective-animal-123.eu-west-1.convex.site
```

> **How to find your site URL:** Take your `CONVEX_URL`, replace `.cloud` with `.site`. If your URL has a region prefix (e.g. `eu-west-1`), keep it.

## Step 4: Seed the Super Admin

The system has no public signup. The first admin account must be seeded. This is a two-step process because Better Auth's admin plugin requires an existing admin to assign roles (chicken-and-egg problem).

### 4a. Create the user account

```bash
npm run init-db
```

This creates a user with:
- Email: `superadmin@politikos.com`
- Password: `ChangeMe!2026`

The script will report that "Set role failed" — this is expected. The user is created but doesn't have a role yet.

You can customize the credentials via environment variables before running:
```bash
SUPER_ADMIN_EMAIL=you@example.com SUPER_ADMIN_PASSWORD=YourPassword123 npm run init-db
```

### 4b. Set the super admin role

Since the admin API requires an existing admin to set roles, we bypass it by writing directly to the Convex database:

**On Linux/macOS:**
```bash
npx convex run seedAdmin:setSuperAdmin '{"email":"superadmin@politikos.com"}'
```

**On Windows (PowerShell):**
```powershell
npx convex run seedAdmin:setSuperAdmin '{\"email\":\"superadmin@politikos.com\"}'
```

> **PowerShell escaping:** Single quotes don't protect double quotes in PowerShell. You must backslash-escape every `"` inside the JSON argument.

You should see:
```
{ success: true, role: "superAdmin" }
```

If you used a custom email in step 4a, replace the email in the command above.

---

## Step 5: Start the Server

In a separate terminal from `npx convex dev`:

```bash
npm run dev
```

The server starts at `http://localhost:3001`. Verify it's running:

```bash
curl http://localhost:3001/health
# {"status":"ok"}
```

---

## Step 6: Verify Everything Works

### Test login
```bash
curl -X POST http://localhost:3001/v1/auth/sign-in/email \
  -H "Content-Type: application/json" \
  -H "Origin: http://localhost:3001" \
  -d '{"email":"superadmin@politikos.com","password":"ChangeMe!2026"}'
```

You should get back a JSON response with the user data and a `set-cookie` header.

### Test submitting a report
```bash
curl -X POST http://localhost:3001/v1/reports \
  -H "Content-Type: application/json" \
  -d '{"category":"TEST","title":"Test report","description":"Setup verification"}'
```

You should get back a `201` with a `trackingCode`.

See `docs/api-reference.md` for the full API documentation and `docs/api-testing.md` for curl examples for every endpoint.

## Troubleshooting

### "CONVEX_URL environment variable is required"

Your `.env.local` file is missing or `npx convex dev` hasn't been run yet. Run `npx convex dev` first (Step 2).

### "You are using the default secret"

You haven't set `BETTER_AUTH_SECRET` on the Convex deployment. Run:
```bash
npx convex env set BETTER_AUTH_SECRET $(node -e "console.log(require('crypto').randomBytes(32).toString('base64'))")
```

### "MISSING_OR_NULL_ORIGIN" on auth requests

Better Auth requires an `Origin` header. Add `-H "Origin: http://localhost:3001"` to your requests. The `init-db` script already handles this.

### "Server misconfigured" on auth endpoints

`CONVEX_SITE_URL` is missing from `.env.local`. See Step 3b.

### init-db says "Set role failed"

This is expected. The admin plugin can't set roles without an existing admin. Follow Step 4b to set the role directly via `npx convex run`.

### Convex run JSON parse error on Windows

PowerShell handles quotes differently. Use backslash-escaped quotes:
```powershell
npx convex run seedAdmin:setSuperAdmin '{\"email\":\"superadmin@politikos.com\"}'
```

### Admin endpoints return 401 even with cookie

Make sure you're including both the `Cookie` header and the `Origin` header in your requests.

### OSM endpoints are slow

The first call to `/v1/osm/upazilas` hits the Overpass API and can take 10-30 seconds. Subsequent calls within 1 hour are cached in memory.
